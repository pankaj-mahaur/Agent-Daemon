03🛠️Install — fresh laptop to daemon-ready

Five steps. ~10 minutes. Zero API keys.

📋 Prerequisites checklist

ToolRequired?Install command
✅ Node.js 22+Requiredwinget install OpenJS.NodeJS.LTS · brew install node
✅ gitRequiredwinget install Git.Git · brew install git

🚀 Step 1 — Install Claude Code

# Mac
brew install --cask claude

# Windows
winget install Anthropic.Claude

# Verify
claude --version

📦 Step 2 — Clone agent-daemon repo

git clone https://github.com/Pankaj-mobiux/Agent-Daemon.git
cd Agent-Daemon

📍 Pick any path you like. Just remember where it lives — npm link next will connect ad to it.

🔗 Step 3 — Register ad globally

cd runtime
npm install
npm link

# Now `ad` works from anywhere
ad --version

✅ Step 4 — Verify with ad doctor

$ ad doctor

  ✓  agent-daemon CLI                  running from .../Agent-Daemon/runtime
  ✓  claude on PATH                    .../claude.exe
  ✓  Auth                              no API key needed
  ✓  better-sqlite3 (episodic memory)  installed
  ✓  Claude Code settings.json         hooks wired
  ✓  constitution/, memory-templates/, skills/, hooks/

agent-daemon: all checks passed.

🎁 Step 5 — Optional add-ons

# qmd — 10x faster skill retrieval (recommended)
npm install -g @tobilu/qmd

# graphify — codebase knowledge graph (optional)
pip install graphifyy
💡Both are auto-detected by ad init in the next step.

04🎬Use it in your project

Two commands and one bootstrap. Done.

1️⃣ Initialize in your project

cd /path/to/your-project

ad init --plan --skills-mode smart   # preview first
ad init                              # apply

4 install modes

ModeWhat it installs
smart (default)🧠 Detects stack (React, FastAPI, Eleventy…) → only relevant skills
all📦 Every skill (229) — heavy but everything available
minimal🪶 Just the always block (bootstrap, orchestrate, skill-author…)
manual📋 Profile-listed skills only (legacy)

2️⃣ What gets created

your-project/
├── 📂 .agent-daemon/
│   ├── 📂 memory/                  ← 7 markdown files (the brain)
│   │   ├── projectbrief.md
│   │   ├── productContext.md
│   │   ├── activeContext.md        ← most updates land here
│   │   ├── systemPatterns.md
│   │   ├── techContext.md
│   │   ├── progress.md
│   │   └── user.md
│   ├── 📂 handoffs/                ← per-session brief
│   └── 📄 sessions.jsonl           ← audit ledger
├── 📂 .claude/skills/              ← project-local skills
├── 📄 AGENTS.md                    ← multi-agent guide
├── 📂 session-logs/                ← journal (gitignored)
└── 📄 CLAUDE.md                    ← managed block appended

3️⃣ Bootstrap memory (one time, ~$0.05)

In your first Claude Code session for the project, type:

"bootstrap the daemon memory using the bootstrap-daemon skill"

Claude scans package.json, README.md, key folders, recent git log → fills 7 memory files with real project context. After this, future sessions are essentially free. 🎉

05🪄Daily magic — what happens during a session

🔄 The session lifecycle

   📱 You launch Claude Code
        ↓
   ⚡ SessionStart hook fires
        ├─ Load constitution (12 cardinal rules)
        ├─ Load memory/*.md (your project context)
        └─ Load recent learnings (cross-session)
        ↓
   🧑‍💻 You work normally — Claude has context
        ├─ Hooks capture file edits, bash runs (silent)
        ├─ Skills auto-trigger from what you say
        └─ Learnings buffered as you go
        ↓
   👋 You say "bye, aaj ka kaam ho gaya"
        ↓
   📝 session-close skill runs (no API key)
        ├─ Update session-logs/
        ├─ Emit digest block
        └─ Write handoff doc
        ↓
   💾 SessionEnd hook → memory updated
        ↓
   🚀 Next session starts smarter

06🛡️Is this Anthropic-ban-safe?

Short answer: yes — per-developer install on own laptop = 100% safe.

Anthropic banned third-party tools like OpenClaw in Feb–Apr 2026 for token arbitrage. agent-daemon does NOT do that.

❌ What Anthropic banned

  • Extracting OAuth tokens from Claude.ai
  • Spoofing the Claude Code harness
  • Subscription auth on VPS / shared server
  • Token arbitrage (sub-priced sub → API workload)

✅ What agent-daemon does

  • Uses Claude Code's documented hooks
  • Uses native Skills convention (~/.claude/skills/)
  • Shells out to the official claude CLI
  • Uses ANTHROPIC_API_KEY (Commercial Terms, optional)
🔒Bottom line: Each Mobiux dev installs on their own laptop with their own Claude Code subscription. Zero TOS issues. Verified 2026-05-21.
